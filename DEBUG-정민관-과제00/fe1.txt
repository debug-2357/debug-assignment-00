 프론트엔드란?

클라이언트 사이드란? 사용자 측에서 처리하는 것
반응형 웹 : 브라우저, 기기, 스크린 사이즈에 따른 지원
웹뷰 구현 : 앱에서 웹을 띄운다.
디자인 구현 : 프론트엔드는 디자인을 구현할 뿐
기능 개발 : 프론트엔드가 하는 일
어플리케이션 최적화 사용자가 최적화된 웹경험을 할 수 있도록.!
다양한 브라우저 지원



프론트엔드가 하는 일 : 1. 데이터를 잘(예쁘게) 사용자에게 보여주기 //쉬움 
  			    2. 데이터(화면)를 조작하기
  			    3. 서버로 데이터를 보내기
  			    4. 서버에서 받은 데이터를 다루기 //어려움 순으로

프론트엔드의 장단점 : 장점 1. 눈에 보인다 //설명하기 좋다 비전공자에게 결과물을 실시간으로 본다.
				  2. 언어에 대한 고민이 적다 다양한 프레임워크 / 수많은 라이브러리
				  3. 혼자 프로젝트를 시작하기 좋다, 백엔드로 옮기기에도 적합하다
				  4. 긴급대응이 없다

			    단점 1. 급변하는 기술 트렌드 수많은 라이브러리와 프레임워크 업데이트
				  2. 중간계
